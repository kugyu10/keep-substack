<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Substack RSS → fetchWithRetry | 外部フィードから取得した `channel.image.url` を `<img src>` に渡す |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Spoofing | `feed.image?.url` | accept | Substack フィードは HTTPS で取得。URL が第三者に差し替えられるリスクは Substack CDN 信頼に依拠。`<img src>` はスクリプト実行しない |
| T-08-02 | Information Disclosure | `<img src={imageUrl}>` | accept | `imageUrl` は Substack 公開 CDN URL。PII を含まない。`alt=""` で装飾画像として扱う |
| T-08-03 | Tampering (XSS) | `<img src={imageUrl}>` | mitigate | `imageUrl` を `<img src>` に直接渡す場合、`javascript:` プロトコルは `<img src>` では実行されない。ただし React の JSX は文字列属性としてエスケープするため追加の XSS リスクなし |
| T-08-04 | Denial of Service | `fetchWithRetry` | accept | 既存の timeout 5秒 + 1回リトライで対応済み（D-03/D-01）。画像取得失敗はフォールバック `<span>` で画面崩れなし |
</threat_model>
