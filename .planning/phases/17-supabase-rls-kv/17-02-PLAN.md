# Plan 17-02: Supabaseクライアントライブラリ基盤

**Phase:** 17 — Supabaseスキーマ + RLS設定 + KVデータ移行
**Plan:** 17-02
**Wave:** 1
**Status:** draft

## Objective

`@supabase/supabase-js` と `@supabase/ssr` をインストールし、3種類のSupabaseクライアント（server / client / admin）を `src/lib/supabase/` 以下に作成する。
アプリからDB接続が成功し、membersテーブルをSELECTできる状態にする。

## Requirements Addressed

- MIGRATE-01: アプリから`@supabase/supabase-js`を使ってDB接続が成功し、membersテーブルをSELECTできる（SC-3）
- MIGRATE-01: 環境変数がVercel本番環境に設定されている（SC-4）

## Pre-implementation Notes

- `@supabase/ssr` はNext.js App Router向けのサーバー/クライアントクッキー管理を提供する
- 接続URLはTransaction Pooler URL（port 6543）必須（D-11）
- サーバーサイドのユーザー確認は`getUser()`のみ使用。`getSession()`は禁止（D-12）
- 公開ISRページ（/、/member/*）ではSupabase Authクライアントを一切使わない（D-13）
- admin.ts（service_role）はサーバーサイド専用。クライアントバンドルに含めてはいけない

## Success Criteria

1. `npm install` 後に `@supabase/supabase-js` と `@supabase/ssr` が `node_modules` に存在すること
2. `src/lib/supabase/server.ts`、`client.ts`、`admin.ts` の3ファイルが存在すること
3. `.env.local.example` に3つの環境変数（NEXT_PUBLIC_SUPABASE_URL、NEXT_PUBLIC_SUPABASE_ANON_KEY、SUPABASE_SERVICE_ROLE_KEY）が記載されていること
4. `npm run build` がエラーなく完了すること

### Task 2: src/lib/supabase/server.ts を作成する

**File:** `src/lib/supabase/server.ts`（新規作成）

Server Components・Route Handlers・Server Actions向けのクライアント。
`@supabase/ssr` の `createServerClient` を使用する。

```typescript
import { createServerClient } from '@supabase/ssr'
import { cookies } from 'next/headers'

export async function createSupabaseServerClient() {
  const cookieStore = await cookies()
  return createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll() {
          return cookieStore.getAll()
        },
        setAll(cookiesToSet) {
          try {
            cookiesToSet.forEach(({ name, value, options }) =>
              cookieStore.set(name, value, options)
            )
          } catch {
            // Server Componentからの呼び出し時は無視
          }
        },
      },
    }
  )
}
```

### Task 4: src/lib/supabase/admin.ts を作成する

**File:** `src/lib/supabase/admin.ts`（新規作成）

Cron・移行スクリプト向けのadminクライアント。
service_role keyを使用するため、**サーバーサイド専用**。

```typescript
import { createClient } from '@supabase/supabase-js'

export function createSupabaseAdminClient() {
  return createClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.SUPABASE_SERVICE_ROLE_KEY!,
    {
      auth: {
        autoRefreshToken: false,
        persistSession: false,
      },
    }
  )
}
```
