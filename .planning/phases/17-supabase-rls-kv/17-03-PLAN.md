# Plan 17-03: KVデータ移行スクリプト

**Phase:** 17 — Supabaseスキーマ + RLS設定 + KVデータ移行
**Plan:** 17-03

**Depends on:** 17-01（スキーマ定義の確認）

## Objective

既存Upstash RedisのKVデータを読み取り、Supabase管理画面のSQL Editorで実行可能なINSERT SQLファイルを生成するスクリプト `scripts/export-to-sql.ts` を作成する。
`tsx scripts/export-to-sql.ts` を実行すると `scripts/output/` 以下に4つのSQLファイルが出力される。

## Requirements Addressed

- MIGRATE-02: 既存Upstash RedisのKVデータをSupabase PostgreSQLに一括マイグレーションできる（SC-5, SC-6, SC-7）

## Pre-implementation Notes

- スクリプトはKVから読む（redis）→ SQLファイルを出力する。Supabaseには直接書き込まない
- 実行コマンド: `tsx scripts/export-to-sql.ts`（dotenvで.env.localを自動ロード）
- 出力先: `scripts/output/members.sql` / `teams.sql` / `member_teams.sql` / `articles.sql`
- 実行順序依存: members.sql → teams.sql → member_teams.sql → articles.sql の順でSQL Editorに貼り付けること
- member_teamsのINSERTはteamsテーブルのname→idサブクエリで生成する（D-16）
- articlesのINSERTはON CONFLICT (link) DO NOTHINGで重複排除する（D-17）
- imageUrl（Substackアイコン）はmembers.image_urlにマッピングする（D-05、D-17）
- `scripts/output/` は `.gitignore` に追加する（生成物は追跡しない）
- 既存の `scripts/` ディレクトリ有無を事前確認してから作成すること

## Success Criteria

1. `tsx scripts/export-to-sql.ts` が正常終了すること
2. `scripts/output/members.sql` に全メンバーのINSERT文が含まれること

### Task 2: scripts/export-to-sql.ts を作成する

**File:** `scripts/export-to-sql.ts`（新規作成）

KVデータを読み取り、INSERT SQLファイルを生成するスクリプト。

```typescript
import 'dotenv/config'
import { mkdir, writeFile } from 'fs/promises'
import { getMembers } from '../src/lib/kvMembers'
import { getArticles } from '../src/lib/kvArticles'

const OUTPUT_DIR = 'scripts/output'

function escapeSql(val: string | null | undefined): string {
  if (val == null) return 'NULL'
  return `'${val.replace(/'/g, "''")}'`
}

async function main() {
  await mkdir(OUTPUT_DIR, { recursive: true })

  const members = await getMembers()
  console.log(`メンバー数: ${members.length}`)

  // --- members.sql ---
  const membersLines = members.map((m) => {
    const name = escapeSql(m.name)
    const substackId = escapeSql(m.substackId)
    const addedAt = escapeSql(m.addedAt)
    // imageUrlはarticlesフェッチ後に更新するためNULLで初期挿入
    return `  (gen_random_uuid(), ${name}, ${substackId}, NULL, ${addedAt})`
  })
  const membersSql = [
    '-- members INSERT（Supabase SQL Editor で実行: 1番目）',
    'INSERT INTO members (id, name, substack_id, image_url, added_at) VALUES',
    membersLines.join(',\n'),
    'ON CONFLICT (substack_id) DO NOTHING;',
  ].join('\n')
  await writeFile(`${OUTPUT_DIR}/members.sql`, membersSql)
  console.log('members.sql を生成しました')

  // --- teams.sql ---
  const allTeamNames = [...new Set(members.flatMap((m) => m.teamNames))].sort()
  console.log(`チーム数: ${allTeamNames.length}`)
  const teamsLines = allTeamNames.map((t) => `  (gen_random_uuid(), ${escapeSql(t)})`)
  const teamsSql = [
    '-- teams INSERT（Supabase SQL Editor で実行: 2番目）',
    'INSERT INTO teams (id, name) VALUES',
    teamsLines.join(',\n'),
    'ON CONFLICT (name) DO NOTHING;',
  ].join('\n')
  await writeFile(`${OUTPUT_DIR}/teams.sql`, teamsSql)
  console.log('teams.sql を生成しました')

  // --- member_teams.sql ---
  const memberTeamsLines: string[] = []
  for (const m of members) {
    for (const teamName of m.teamNames) {
      memberTeamsLines.push(
        `INSERT INTO member_teams (member_id, team_id)\n` +
        `  SELECT m.id, t.id FROM members m, teams t\n` +
        `  WHERE m.substack_id = ${escapeSql(m.substackId)} AND t.name = ${escapeSql(teamName)}\n` +
        `  ON CONFLICT DO NOTHING;`
      )
    }
  }
  const memberTeamsSql = [
    '-- member_teams INSERT（Supabase SQL Editor で実行: 3番目）',
    memberTeamsLines.join('\n'),
  ].join('\n')
  await writeFile(`${OUTPUT_DIR}/member_teams.sql`, memberTeamsSql)
  console.log('member_teams.sql を生成しました')

  // --- articles.sql + members.image_url 更新 ---
  const articlesLines: string[] = []
  const imageUrlUpdates: string[] = []

  for (const m of members) {
    const feed = await getArticles(m.substackId)

    // imageUrlをmembers.image_urlに反映（UPDATEで後追い）
    if (feed.imageUrl) {
      imageUrlUpdates.push(
        `UPDATE members SET image_url = ${escapeSql(feed.imageUrl)}\n` +
        `  WHERE substack_id = ${escapeSql(m.substackId)};`
      )
    }

    for (const item of feed.items) {
      if (!item.link) continue
      const substackId = escapeSql(m.substackId)
      const title = escapeSql(item.title)
      const link = escapeSql(item.link)
      const pubDate = item.isoDate
        ? `'${item.isoDate}'::TIMESTAMPTZ`
        : item.pubDate
          ? escapeSql(item.pubDate)
          : 'NULL'
      articlesLines.push(
        `  (gen_random_uuid(), ${substackId}, ${title}, ${link}, ${pubDate})`
      )
    }
  }

  const articlesSql = [
    '-- articles INSERT（Supabase SQL Editor で実行: 4番目）',
    ...(imageUrlUpdates.length > 0
      ? ['-- members.image_url の更新', ...imageUrlUpdates, '']
      : []),
    'INSERT INTO articles (id, substack_id, title, link, pub_date) VALUES',
    articlesLines.join(',\n'),
    'ON CONFLICT (link) DO NOTHING;',
  ].join('\n')
  await writeFile(`${OUTPUT_DIR}/articles.sql`, articlesSql)

  const totalArticles = members.reduce(
    (sum, _m, i) => sum,
    articlesLines.length
  )
  console.log(`articles.sql を生成しました（${articlesLines.length} 件）`)
  console.log('\n完了。実行順序: members.sql → teams.sql → member_teams.sql → articles.sql')
}

main().catch((err) => {
  console.error(err)
  process.exit(1)
})
```

注意:
- `dotenv/config` で `.env.local` の `UPSTASH_REDIS_REST_URL` / `UPSTASH_REDIS_REST_TOKEN` を自動ロードする
- `getMembers()` / `getArticles()` は既存のKV関数をそのまま再利用する
- スクリプト自体はSupabaseへの書き込みを行わない（SQL生成のみ）
