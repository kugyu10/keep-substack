## Summary

このフェーズの変更は3ファイル、最小限の修正で完結する。`src/app/page.tsx` と `src/app/member/[substackId]/page.tsx` の先頭に `export const revalidate = 300` を1行追加し、`src/lib/fetchFeed.ts` の `fetchAllFeedsCached()` 内部実装をハイブリッドフェッチに変更する。

`fetchAllFeedsCached()` の変更は、各メンバーに対して `fetchWithRetry`（ライブRSS）と `getArticles`（KV過去記事）を並列実行し、`FeedItem.link` で dedupe してマージするパターンに書き換える。シグネチャは変更しないため、呼び出し元への影響はゼロ。

Next.js 15 では `export const revalidate = 300` をページファイルのトップレベルに配置することで、そのルートセグメント全体が ISR として動作する。外部フェッチ（`rss-parser` 経由）は Next.js の `fetch()` API を使用しないため、ルートセグメントの `revalidate` 値がキャッシュ周期を制御する。

**Primary recommendation:** `fetchAllFeedsCached()` 内で `Promise.allSettled` を使い、各メンバーのライブRSS + KV を並列取得し link で dedupe してマージする。ページには `export const revalidate = 300` を追加するだけ。

### Pattern 2: `Promise.allSettled` によるハイブリッドフェッチと dedupe マージ

**What:** 各メンバーに対して RSS と KV を並列取得し、失敗を隔離しつつマージする。
**When to use:** 外部サービス（RSS）が失敗しても KV フォールバックで表示を維持したい場合。

```typescript
// [VERIFIED: 既存 fetchFeed.ts + kvArticles.ts のパターンを踏襲]
// saveArticles (kvArticles.ts) と同じ link ベース dedupe ロジックを使用

export async function fetchAllFeedsCached(members: Member[]): Promise<MemberFeedResult[]> {
  const results = await Promise.allSettled(
    members.map((m) => Promise.all([
      fetchWithRetry(`https://${m.substackId}.substack.com/feed`),
      getArticles(m.substackId),
    ]))
  )
  return members.map((member, i) => {
    const result = results[i]
    if (result.status === 'rejected') {
      return { member, items: [], imageUrl: undefined }
    }
    const [live, kv] = result.value
    // imageUrl: ライブRSS優先、なければKV
    const imageUrl = live.imageUrl ?? kv.imageUrl
    // dedupe: link で重複除去、ライブRSSが優先（先に追加されるため）
    const seenLinks = new Set<string | undefined>()
    const merged: FeedItem[] = []
    for (const item of [...live.items, ...kv.items]) {
      if (item.link && seenLinks.has(item.link)) continue
      seenLinks.add(item.link)
      merged.push(item)
    }
    // isoDate 降順ソート
    merged.sort((a, b) => {
      if (!a.isoDate) return 1
      if (!b.isoDate) return -1
      return b.isoDate.localeCompare(a.isoDate)
    })
    return { member, items: merged, imageUrl }
  })
}
```

### Pattern 3: imageUrl の優先順位

**What:** ライブ RSS の `feed.image?.url` を優先し、なければ KV に保存された imageUrl を使う。
**Why:** ライブ RSS は常に最新のアイコン URL を持つ可能性があるため優先。KV は過去フェッチ時の値で安定的なフォールバック。

```typescript
// [VERIFIED: src/lib/fetchFeed.ts の fetchWithRetry 戻り値と kvArticles.ts の StoredFeed スキーマ確認済み]
const imageUrl = live.imageUrl ?? kv.imageUrl
```

### Anti-Patterns to Avoid

- **`fetchAllFeedsCached()` 内で KV に書き込まない:** D-03 による制約。Cron ジョブの責務を侵食する。
- **シグネチャ変更:** D-04 により禁止。呼び出し元（page.tsx）への修正が増える。
- **link が undefined の記事を dedupe キーにする:** `item.link &&` チェックを必ず入れる。`undefined` 同士が重複と見なされないよう注意（`Set.has(undefined)` は true になる）。
- **RSS フェッチを直列実行:** `Promise.allSettled` で並列実行する。直列にすると全メンバー分の待機が積み重なる。

## Common Pitfalls

### Pitfall 1: `export const revalidate` の配置場所

**What goes wrong:** default export の後や関数内に書いても ISR が有効にならない。
**Why it happens:** Next.js はルートセグメント設定をファイルの静的解析で読むため、トップレベルの export が必要。
**How to avoid:** import 文の直後、default export の前に書く。
**Warning signs:** ローカルでは動作するが Vercel でキャッシュが効かない。

### Pitfall 2: link が undefined の記事が dedupe を壊す

**What goes wrong:** `seenLinks.add(undefined)` を複数回実行すると、link のない記事が全て1件に集約される。
**Why it happens:** `Set` は `undefined` を値として保持するため、link 未設定記事が最初の1件以外すべて除外される。
**How to avoid:** `if (item.link && seenLinks.has(item.link)) continue` と `seenLinks.add(item.link)` の両方に `item.link &&` ガードを入れる。

既存の `saveArticles` も同じアプローチ: [VERIFIED: src/lib/kvArticles.ts L21-22]
```typescript
const toAdd = newItems.filter((item) => item.link && !existingLinks.has(item.link))
```

### Pitfall 3: Next.js 15 のデフォルトキャッシュ動作の変更

**What goes wrong:** Next.js 15 では fetch リクエストのデフォルトが no-cache（`autoNoCache`）に変わった。`export const revalidate = 300` を設定しても、個別 fetch が no-cache を明示していれば上書きされない。
**Why it happens:** Next.js 15 の `autoNoCache` フラグが true の場合、`revalidate` は 0（動的レンダリング）になる。[VERIFIED: Context7 /vercel/next.js patch-fetch.ts]
**How to avoid:** `fetchAllFeedsCached` は `fetch()` API ではなく `rss-parser` の `parseURL` を使うため、この問題は **本プロジェクトでは非該当**。`rss-parser` は内部的に Node.js の http/https を使うため、Next.js の fetch キャッシュ機構の影響を受けない。`export const revalidate = 300` がルートセグメント全体の再生成間隔を制御する。

### Pitfall 4: `Promise.allSettled` と `Promise.all` の使い分け

**What goes wrong:** 内側の `Promise.all([fetchWithRetry(...), getArticles(...)])` が rejected になった場合、外側の `Promise.allSettled` で status === 'rejected' として捕捉できる。問題は内側に `Promise.all` を使うと、RSS と KV のどちらか一方が失敗しただけでそのメンバー全体が rejected になること。
**Why it happens:** `Promise.all` は最初の reject で全体が reject される。
**How to avoid:** 内側も `Promise.allSettled` にするか、個別に try/catch でフォールバックを設定する。**ただし D-02 の仕様では「ライブ RSS 失敗時は KV のみで返す」ため**、内側を `Promise.allSettled` にして RSS 失敗を個別ハンドリングするのが正確な実装。

```typescript
// 内側も allSettled で個別ハンドリング
const [liveResult, kvResult] = await Promise.allSettled([
  fetchWithRetry(`https://${m.substackId}.substack.com/feed`),
  getArticles(m.substackId),
])
const live = liveResult.status === 'fulfilled' ? liveResult.value : { items: [], imageUrl: undefined }
const kv = kvResult.status === 'fulfilled' ? kvResult.value : { items: [] }
```

### fetchAllFeedsCached ハイブリッド実装

```typescript
// [VERIFIED: 既存コード fetchFeed.ts, kvArticles.ts のパターンを基に構成]
// D-02: ライブRSS + KV マージ, D-03: KV書き込みなし, D-04: シグネチャ維持

export async function fetchAllFeedsCached(members: Member[]): Promise<MemberFeedResult[]> {
  const results = await Promise.allSettled(
    members.map(async (m) => {
      const [liveResult, kvResult] = await Promise.allSettled([
        fetchWithRetry(`https://${m.substackId}.substack.com/feed`),
        getArticles(m.substackId),
      ])
      const live = liveResult.status === 'fulfilled'
        ? liveResult.value
        : { items: [], imageUrl: undefined }
      const kv = kvResult.status === 'fulfilled'
        ? kvResult.value
        : { items: [] as FeedItem[] }
      return { live, kv }
    })
  )

  return members.map((member, i) => {
    if (results[i].status === 'rejected') {
      return { member, items: [], imageUrl: undefined }
    }
    const { live, kv } = (results[i] as PromiseFulfilledResult<{ live: { items: FeedItem[]; imageUrl?: string }, kv: { items: FeedItem[]; imageUrl?: string } }>).value
    const imageUrl = live.imageUrl ?? kv.imageUrl
    const seenLinks = new Set<string>()
    const merged: FeedItem[] = []
    for (const item of [...live.items, ...kv.items]) {
      if (item.link && seenLinks.has(item.link)) continue
      if (item.link) seenLinks.add(item.link)
      merged.push(item)
    }
    merged.sort((a, b) => {
      if (!a.isoDate) return 1
      if (!b.isoDate) return -1
      return b.isoDate.localeCompare(a.isoDate)
    })
    return { member, items: merged, imageUrl }
  })
}
```
