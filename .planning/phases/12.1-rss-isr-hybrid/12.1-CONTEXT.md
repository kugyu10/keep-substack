# Phase 12.1: rss-isr-hybrid - Context

**Gathered:** 2026-05-12
**Status:** Ready for planning

<domain>
## Phase Boundary

現在 `fetchAllFeedsCached()` は KV のみを参照する（Cron が1日1回 KV を更新）。
新規投稿が表示されるまで最大24時間かかる問題がある。

このフェーズでは **ISR（revalidate=300）+ KV ハイブリッドアーキテクチャ** を導入し、
投稿後5分以内にトップページ・メンバー詳細ページへ反映させる。

変更スコープ:
- `src/app/page.tsx` に `export const revalidate = 300` を追加（ISR）
- `src/app/member/[substackId]/page.tsx` に `export const revalidate = 300` を追加
- `fetchAllFeedsCached()` を変更してライブRSSとKV過去記事のマージを行う

</domain>

<decisions>
## Implementation Decisions

### ISR キャッシュ設定
- **D-01:** `revalidate = 300`（5分）を page.tsx と member ページの両方に設定する。
  Next.js の `export const revalidate` を使用（next.config.ts 変更不要）。

### フェッチ戦略（ハイブリッド）
- **D-02:** `fetchAllFeedsCached()` はライブ RSS フェッチ（`fetchWithRetry`）+ KV 既存データのマージ結果を返す。
  - ライブ RSS から最新記事を取得（最大20件程度）
  - KV から過去累積記事を取得
  - article.link URL で dedupe してマージ（重複は1件にまとめる）
  - ライブ RSS 失敗時は KV のみにフォールバック（既存動作維持）

### KV 保存タイミング
- **D-03:** `fetchAllFeedsCached()` 内ではライブフェッチ結果を KV に保存しない（Cron の責務を維持）。
  ページ表示時の保存は行わない（YAGNI）。

### シグネチャ互換
- **D-04:** `fetchAllFeedsCached(members: Member[])` のシグネチャを変更しない。
  呼び出し元（page.tsx, member/[substackId]/page.tsx）の変更は revalidate 追加のみ。

### Claude's Discretion
- マージロジックの実装場所: `fetchAllFeedsCached()` 内（kvArticles.ts の saveArticles とは独立）
- `Promise.allSettled` を使用して1件失敗が全体に影響しない設計を維持
- ライブ RSS + KV マージ後のソート: isoDate 降順

</decisions>

<code_context>
## Existing Code Insights

### 現在の fetchAllFeedsCached（変更前）
```typescript
export async function fetchAllFeedsCached(members: Member[]): Promise<MemberFeedResult[]> {
  const results = await Promise.allSettled(
    members.map((m) => getArticles(m.substackId))
  )
  return members.map((member, i) => {
    const data = results[i].status === 'fulfilled' ? results[i].value : { items: [] }
    return { member, items: data.items, imageUrl: data.imageUrl }
  })
}
```

### 変更後のイメージ
- 各メンバーに対して `fetchWithRetry` + `getArticles` を並列実行
- ライブ RSS のアイテムと KV アイテムを link で dedupe してマージ
- ライブ RSS 失敗時は KV のみで返す（フォールバック）

### Established Patterns
- `Promise.allSettled` で失敗を隔離（Phase 10 確立済み）
- `fetchWithRetry` は既に実装済み（src/lib/fetchFeed.ts）
- `getArticles` は既に実装済み（src/lib/kvArticles.ts）
- KV の dedupe は article.link 一致で判定（Phase 10 確立済み）

### Integration Points
- `fetchAllFeedsCached` の内部実装変更のみ（シグネチャ変更なし）
- page.tsx と member ページに `export const revalidate = 300` を追加

</code_context>

<specifics>
## Specific Ideas

- ISR revalidate=300 により、Vercel は5分ごとにページを再生成する
- ライブ RSS は直近20件程度しか取得できないが、KV に過去分が蓄積されているため歴史カレンダーは維持される
- `fetchWithRetry` の URL パターン: `https://${substackId}.substack.com/feed`（Phase 4 確立済み）
- マージ後のソート: isoDate 降順（カレンダー表示のため）
- imageUrl はライブ RSS 優先、なければ KV のものを使用

</specifics>
