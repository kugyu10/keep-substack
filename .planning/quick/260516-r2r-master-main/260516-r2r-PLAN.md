<task type="auto">
  <name>Task 1: ローカルブランチを master から main にリネームし、リモートに push する</name>
  <files>（Git ブランチ操作のみ — ファイル変更なし）</files>
  <action>
    以下の Git コマンドを順に実行する。

    1. ローカルブランチをリネーム:
       git branch -m master main

    2. main を origin に push し、upstream を設定:
       git push -u origin main

    3. リモートの HEAD を main に変更（GitHub API 経由）:
       gh repo edit --default-branch main

    4. リモートの master を削除:
       git push origin --delete master

    注意:
    - `gh repo edit --default-branch main` は GitHub 上のデフォルトブランチを変更する。これを実行してから origin/master を削除しないと、GitHub がエラーを返す場合がある。必ずステップ3→4の順で実行すること。
    - gh CLI が未認証の場合は `gh auth login` を先に実行する。
  </action>
  <verify>
    <automated>git branch -a | grep -E "main|master" && git remote show origin | grep "HEAD branch"</automated>
  </verify>
  <done>
    - `git branch -a` の出力に remotes/origin/main が存在する
    - `git remote show origin` の "HEAD branch" が main を示す
    - remotes/origin/master が出力に含まれない
    - ローカルの現在ブランチが main である（git branch --show-current が "main" を返す）
  </done>
</task>
