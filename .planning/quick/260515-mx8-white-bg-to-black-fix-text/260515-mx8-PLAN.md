  <action>
`:root` ブロックの `--background` と `--foreground` を黒背景に合わせた値に変更する。

変更内容：
```css
:root {
  --background: #0a0a0a;   /* 白 #ffffff → 黒 #0a0a0a */
  --foreground: #ededed;   /* 暗 #171717 → 明 #ededed */
}
```

`@media (prefers-color-scheme: dark)` ブロックは不要になるが、残しても動作に影響はないため削除して簡潔にする（KISS）。

削除対象：
```css
@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}
```
  </action>

  <done>globals.css の :root に --background: #0a0a0a が設定されている</done>

<task type="auto">
  <name>Task 2: WeeklyHeatmapGrid・HeatmapRow の黒背景対応</name>
  <files>src/components/WeeklyHeatmapGrid.tsx, src/components/HeatmapRow.tsx</files>
  <action>
## WeeklyHeatmapGrid.tsx

週ナビゲーションボタンのhover色が `hover:text-gray-800 hover:bg-gray-100` で、黒背景にすると hover 時に暗色になり視認性が悪い。明るい色に変更する。

変更：
- `hover:text-gray-800 hover:bg-gray-100` → `hover:text-white hover:bg-gray-700`

日付ヘッダーの `text-gray-500` は黒背景でも視認できるが、土日の色（`text-cyan-500`・`text-orange-500`）はそのままでOK。

## HeatmapRow.tsx

行区切り線 `border-gray-100` は白背景用（明るいグレー）。黒背景では目立ちすぎず溶けすぎない `border-gray-800` に変更する。

変更：
- `border-b border-gray-100` → `border-b border-gray-800`

メンバー名テキストはデフォルト foreground 色を継承するため変更不要。
`text-gray-400`（シェブロン・計ラベル）は黒背景でも視認できるためそのまま。
  </action>
  <verify>npm run build 2>&1 | tail -5 でビルドエラーがないこと</verify>
  <done>
- WeeklyHeatmapGrid の週ナビhover が hover:bg-gray-700 になっている
- HeatmapRow の行区切り線が border-gray-800 になっている
  </done>
</task>
