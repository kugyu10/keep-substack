# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-05-08
**Phases:** 1-3
**Total Plans:** 6

## Overview

Substack継続仲間コミュニティ向けの、メンバーの記事公開頻度をカレンダーUIで可視化するWebアプリのMVP。RSSフィード取得基盤の構築から、月別カレンダーUI、全メンバーダッシュボード、個人詳細ページまでを3フェーズで完成させ、Vercelにデプロイして公開。

**Deployed URL:** https://keep-substack.vercel.app/

## Phases

**Goal**: RSSフィードからメンバーの記事データを取得・解析し、ISRでキャッシュされたページとしてVercelで配信できる状態にする
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01-PLAN.md — Next.jsプロジェクト初期化 + メンバー設定JSON + RSS取得ロジック
- [x] 01-02-PLAN.md — ISR設定 + 最小表示ページ + Vercelデプロイ

**Details:**
- Next.js 16.2.6 (App Router + TypeScript + Tailwind CSS) プロジェクト基盤構築
- rss-parser@3.13.0 による並列フィード取得（Promise.allSettled、タイムアウト5秒、リトライ付き）
- unstable_cache + REVALIDATE_SECONDS 環境変数によるISR制御
- export const dynamic = 'force-static' でキャッシュ可変revalidateを実現
- Vercel デプロイ完了（REVALIDATE_SECONDS=300）

## Milestone Summary

**Key Decisions:**

- ISR（force-static + unstable_cache）でrevalidate可変制御 — export const revalidate はリテラルのみ有効
- @types/rss-parser は不要（rss-parser本体にindex.d.tsが同梱）
- Map シリアライズ不可のため配列変換してServer→Client渡し
- Tailwind 動的クラスはJIT非検知のためstyle propを使用
- MiniCalendarはCalendarGridとは別のServer Componentとして独立（KISS原則）
- dynamicParams=false で未知の substackId を自動404（セキュリティ対応）

**Issues Resolved:**

- create-next-app が既存ファイルのある./ への直接実行を拒否 → 一時ディレクトリ方式で解決
- .gitignore の .env* パターンが .env.example を対象に → git add --force で追加
- ツールチップとセル間のホバー離脱バグ → 透明パディングで修正

**Issues Deferred:**

- MOT-01: 連続投稿日数（ストリーク）表示 → v2
- MOT-02: 月間投稿数サマリー → v2
- MOT-03: 年間ヒートマップ → v2
- MGT-01: ブラウザ上でフィードURLを管理する管理画面 → v2

**Technical Debt Incurred:**

- なし（KISS/YAGNI原則に従いシンプルに実装完了）
