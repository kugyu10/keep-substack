# Milestone v1.3: Data Persistence + Multi-Team

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 10-12.1
**Total Plans:** 4

## Overview

Vercel Cron + KV累積保存による過去記事永続化、多対多チーム所属、シークレットチーム機能、ISR + KVハイブリッドアーキテクチャを実装。データの信頼性と鮮度を大幅に改善。

## Phases

### Phase 10: Cron + KV記事永続化

**Goal:** Vercel Cronによる1日1回のRSS取得とKV累積保存で、過去記事が消えずカレンダー・ヒートマップに利用できる状態にする
**Depends on:** Phase 9 (v1.2完了済み)
**Requirements:** PERSIST-01, PERSIST-02, PERSIST-03
**Plans:** 1 plan

Plans:
- [x] 10-01-PLAN.md — kvArticles新設・fetchFeed KV移行・Cronエンドポイント・addMemberAction初回取得

**Details:**
- kvArticles.ts 新設（getArticles / saveArticles）
- GET /api/cron — CRON_SECRET Bearer認証付きCronエンドポイント
- vercel.json で 0 20 * * * スケジュール設定
- fetchAllFeedsCached を unstable_cache → KV直接読み込みに移行（シグネチャ維持）
- addMemberAction 登録直後に初回フィード取得・KV保存

### Phase 11: チーム多対多所属

**Goal:** 1人のメンバーが複数チームに所属でき、管理画面で設定・チームフィルターで正しく絞り込める状態にする
**Depends on:** Phase 10
**Requirements:** TEAM-01, TEAM-02, TEAM-03
**Plans:** 1 plan

Plans:
- [x] 11-01-PLAN.md — types/KV後方互換/page/AdminUI/actions/memberページ 全7ファイル変更

**Details:**
- Member.teamName: string → teamNames: string[] に型変更
- KV後方互換フォールバック: m.teamNames ?? (m.teamName ? [m.teamName] : [])
- 管理画面カンマ区切りテキスト入力 → Server Actions で string[] 変換
- 複数チーム戻りリンク: flex gap-3 横並び + encodeURIComponent

### Phase 12: chameleon-hidden-team

**Goal:** シークレットチーム "chameleon" を定義し、所属メンバーを All ビューおよびチームタブから非表示にする
**Depends on:** Phase 11
**Plans:** 1 plan

Plans:
- [x] 12-01-PLAN.md — HIDDEN_TEAM 定数追加・teams フィルター・All ビュー filteredMembers フィルター

**Details:**
- src/lib/types.ts に `export const HIDDEN_TEAM = 'chameleon'` 追加
- teams 生成に `.filter(t => t !== HIDDEN_TEAM)` 追加（タブ非表示）
- filteredMembers に `.filter(m => !m.teamNames.includes(HIDDEN_TEAM))` 追加（All ビュー除外）
- /?team=chameleon 直打ち・管理画面は通常表示を維持

### Phase 12.1: rss-isr-hybrid (INSERTED)

**Goal:** RSS ISR復活 + KVのハイブリッドアーキテクチャ — 投稿後5分以内に反映されるよう、ISR（revalidate=300）でRSS直接フェッチとKV過去記事をマージして表示する
**Depends on:** Phase 11
**Plans:** 1 plan

Plans:
- [x] 12.1-01-PLAN.md — fetchAllFeedsCached ハイブリッド実装・page.tsx と member ページに revalidate=300 追加

**Details:**
- fetchAllFeedsCached を ライブRSS(fetchWithRetry) + KV(getArticles) のマージ実装に書き換え
- 内側 Promise.allSettled でライブRSS失敗時はKVのみにフォールバック
- link dedupe: `if (item.link && seenLinks.has(item.link)) continue` — undefined link記事はすべて含む
- page.tsx と member/[substackId]/page.tsx に `export const revalidate = 300` 追加
- 最大24時間 → 最大5分の反映遅延改善
