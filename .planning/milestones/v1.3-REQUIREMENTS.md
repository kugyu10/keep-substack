## v1.3 Requirements

### 記事永続化 (Persist)

- [x] **PERSIST-01**: Vercel Cron（1日1回）でRSSフィードを取得しKVに記事データを累積保存できる
  - **Outcome:** Phase 10 で実装。GET /api/cron エンドポイント + vercel.json スケジュール (0 20 * * *)。CRON_SECRET Bearer認証済み。
- [x] **PERSIST-02**: メンバー登録時に初回フィード取得を実行してKVに保存する
  - **Outcome:** Phase 10 で実装。addMemberAction 内で fetchWithRetry + saveArticles を実行。
- [x] **PERSIST-03**: 累積済み記事データをカレンダービュー・ヒートマップで使用できる
  - **Outcome:** Phase 10 で実装。fetchAllFeedsCached を KV直接読み込みに移行（unstable_cache廃止）。Phase 12.1 で ISR + ライブRSSハイブリッドに進化。

### チーム多対多 (Team)

- [x] **TEAM-01**: 1人のメンバーが複数チームに所属できる（teamNames: string[]）
  - **Outcome:** Phase 11 で実装。Member.teamName → teamNames: string[] に型変更。KV後方互換フォールバック付き。
- [x] **TEAM-02**: 管理画面でメンバーの所属チームをカンマ区切りで複数設定・更新できる
  - **Outcome:** Phase 11 で実装。AdminMemberList のカンマ区切り入力 + Server Actions で string[] 変換。
- [x] **TEAM-03**: チームフィルターで複数チーム所属のメンバーがどのチームでも表示される
  - **Outcome:** Phase 11 で実装。flatMap + includes パターンで多対多フィルタリング対応。
